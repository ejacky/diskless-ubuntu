# Diskless Multi-OS Platform

基于 **iPXE + iSCSI + LVM Thin Snapshot** 的开源无盘系统平台，支持 Linux 与 Windows 多系统网络启动，每次重启自动还原系统盘，同时支持用户数据持久化保存。

> **当前聚焦**：1~5 台客户端，千兆网络，单服务端部署。

---

## 核心特性

| 特性 | 说明 |
|------|------|
| **多系统支持** | Ubuntu、Debian、Windows 10/11 等，开机菜单按需切换 |
| **系统盘还原** | 基于 LVM Thin Snapshot，重启后系统盘自动回到干净状态 |
| **用户数据持久化** | 系统盘与用户数据盘分离，文档/配置重启后保留（可选） |
| **无盘运行** | 客户端无需硬盘，PXE 网络启动，系统盘运行在服务端 iSCSI 存储上 |
| **全开源** | 基于 iPXE、LVM、Linux-IO Target、QEMU 等开源组件 |

## 架构概览

```
┌──────────────────────────────────────────────────────────────────────┐
│                           服务端 (单台，千兆)                          │
│                                                                      │
│  ┌─────────────┐   ┌─────────────┐   ┌──────────────────────────┐  │
│  │  iPXE 引导   │   │ iSCSI Target│   │   LVM Thin Pool          │  │
│  │ (DHCP+TFTP+ │   │ (LIO/       │   │ ┌─────────┐ ┌─────────┐  │  │
│  │  HTTP)      │   │  targetcli) │   │ │ golden  │ │ client  │  │  │
│  └──────┬──────┘   └──────┬──────┘   │ │ images  │ │snapshot │  │  │
│         │                 │          │ │ (ro)    │ │ (rw)    │  │  │
│         └─────────────────┼──────────┘ └─────────┘ └─────────┘  │  │
│                           │          ┌─────────────┐             │  │
│                           │          │ user-data   │             │  │
│                           │          │ (persistent │             │  │
│                           │          │  thin LV)   │             │  │
│                           │          └─────────────┘             │  │
│                           ▼                                        │
│                  ┌─────────────────┐                               │
│                  │   Boot API      │                               │
│                  │ (按MAC动态生成  │                               │
│                  │  iPXE脚本+快照) │                               │
│                  └─────────────────┘                               │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼  PXE / iSCSI / NFS
┌──────────────────────────────────────────────────────────────────────┐
│                          客户端（无盘工作站）x 1~5                     │
│                                                                      │
│   ① iPXE ROM → DHCP 获取 IP + 引导脚本 URL                           │
│   ② HTTP 下载动态 iPXE 脚本，显示多系统菜单                            │
│   ③ 选择系统 → 服务端创建系统盘 snapshot + 挂载用户数据盘              │
│   ④ iSCSI 连接系统盘 + 用户数据盘 → 启动操作系统                       │
│   ⑤ 系统运行：写入系统盘 → snapshot (重启丢弃)                        │
│                 写入用户数据盘 → 持久化 thin LV (永久保留)              │
└──────────────────────────────────────────────────────────────────────┘
```

**关键设计**：系统盘与用户数据盘物理分离。
- **系统盘**：thin snapshot，母盘只读，客户端可写，重启后删除重建 → **纯还原**
- **用户数据盘**：独立的 thin LV（非 snapshot），不删除，长期保留 → **持久化**

## 快速对比：当前 vs 目标

| 维度 | 当前 (`DISKLESS-GUIDE.md`) | 目标架构 (1~5台) |
|------|---------------------------|-----------------|
| 启动器 | PXELINUX / GRUB TFTP | **iPXE**（支持脚本、HTTP、iSCSI sanboot） |
| 菜单 | 静态配置文件 | **动态生成**（按 MAC + 用户选择） |
| 系统数 | 单 Ubuntu | **多系统**（Ubuntu / Windows / ...） |
| 存储 | 单 fileio LUN | **LVM Thin Pool + Snapshot** |
| 还原 | 无 | **系统盘纯还原**，**用户数据可选持久化** |
| 网络 | 千兆单链路 | **千兆（当前场景）** |
| 数据分离 | 无 | **系统盘与用户数据盘独立** |

## 两种使用模式

| 模式 | 系统盘 | 用户数据盘 | 适用场景 |
|------|--------|-----------|----------|
| **模式 A：全还原** | snapshot，重启丢弃 | 不挂载 | 网吧、公共机房 |
| **模式 B：系统还原 + 数据保留** | snapshot，重启丢弃 | 独立持久化 LV，长期保留 | 学校机房、个人工作室 |

> 用户数据盘的持久化是**可选配置**，可在 Boot API 中按客户端或按系统开启/关闭。
